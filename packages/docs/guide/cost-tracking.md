# Cost & Token Tracking

Flightdeck tracks token usage and costs across all agents, tasks, and sessions. This data powers the dashboard displays, budget controls, and analytics.

## How Token Tracking Works

Every API call made by an agent reports cumulative token counts via ACP. Flightdeck uses **delta attribution** to assign tokens to specific tasks:

1. Agent reports cumulative input/output tokens
2. Flightdeck computes the delta: `newTokens = cumulativeNew âˆ’ lastSeenCumulative`
3. The delta is attributed to the agent's current DAG task
4. Records are persisted to the `task_cost_records` table with atomic upserts

This means every token is attributed to exactly one agent, one task, one lead, and one project â€” enabling drill-down analysis at any level.

### Cost Calculation

Token costs use these rates:

| Token Type | Rate |
|------------|------|
| Input tokens | $3.00 per 1M tokens |
| Output tokens | $15.00 per 1M tokens |

Total spend = `inputTokens Ã— $3e-6 + outputTokens Ã— $15e-6`

### Token Types

Each record tracks four token categories:

| Category | Description |
|----------|-------------|
| **Input tokens** | Tokens sent to the model (prompts, context) |
| **Output tokens** | Tokens generated by the model (responses) |
| **Cache read tokens** | Tokens served from prompt cache (reduced cost) |
| **Cache write tokens** | Tokens written to prompt cache |

## Attribution Model

Every token usage record is attributed to:

| Level | Field | Description |
|-------|-------|-------------|
| **Agent** | `agentId` | Which agent made the API call |
| **Task** | `dagTaskId` | Which DAG task the agent was working on |
| **Lead** | `leadId` | Which lead session the agent belongs to |
| **Project** | `projectId` | Which project the work is for |

This enables precise drill-down: _"Project X cost $12, of which $8 was the developer working on task `implement-auth`."_

## Data Sources & Fallback

Token data comes from two sources with automatic fallback:

### Live Agent Data (Active Sessions)

When agents are running, token counts come directly from agent objects in memory:

```
agent.inputTokens  â†’ accumulated from ACP usage reports
agent.outputTokens â†’ accumulated from ACP usage reports
```

This is real-time and updates with every API call the agent makes.

### Persisted Database Data (Historical Sessions)

When a session ends, token data is persisted to `task_cost_records`. For inactive sessions, the UI falls back to the database:

```
GET /costs/by-agent?projectId=...  â†’ reads from task_cost_records
```

The UI automatically detects which source to use â€” if live agent token counts sum to zero, it falls back to database records.

## Budget Controls

Set a spending budget to prevent runaway costs:

```yaml
budget:
  limit: 50.00          # Dollar cap (null = unlimited)
  thresholds:
    warn: 0.7            # 70% â€” show warning in UI
    alert: 0.9           # 90% â€” show critical alert
    pause: 1.0           # 100% â€” pause all agents
```

### Budget Thresholds

| Threshold | Level | Action |
|-----------|-------|--------|
| **70%** | Warning | Yellow indicator in the dashboard |
| **90%** | Alert | Red alert, notification sent |
| **100%** | Pause | All agents are paused automatically |

Set `limit: null` to disable budget enforcement entirely.

### Per-Project Budgets

Budgets can be scoped to individual projects:

```
POST /budget
{ "limit": 25.00, "projectId": "proj-abc" }
```

Check budget status at any time:

```
GET /budget?projectId=proj-abc
â†’ { "limit": 25.00, "currency": "USD", "currentSpend": 12.34, "utilization": 0.49, "paused": false }
```

### Token Budget Optimizer

The `TokenBudgetOptimizer` allocates context-window token budgets across agents proportional to priority and efficiency:

- `allocate(agentId, priority: 1â€“10)` â€” register agent with initial allocation
- `recordUsage(agentId, tokensUsed)` â€” track consumption
- `updateEfficiency(agentId, efficiency: 0.1â€“2.0)` â€” adjust priority
- Automatic rebalancing when agents join or leave the pool

## API Endpoints

### Cost Endpoints

All under `/api/costs/`:

| Endpoint | Description |
|----------|-------------|
| `GET /costs/by-agent` | Token totals per agent. Optional `?projectId=` filter. |
| `GET /costs/by-task` | Token totals per DAG task. Optional `?leadId=` or `?projectId=` filter. |
| `GET /costs/agent/:agentId` | Task-level cost breakdown for a specific agent. |
| `GET /costs/by-project` | Aggregate totals per project. |
| `GET /costs/by-session` | Per-session costs. Requires `?projectId=` parameter. |

### Budget Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/budget` | GET | Budget status. Optional `?projectId=` scope. |
| `/budget` | POST | Set/update budget: `{ limit, thresholds, projectId }` |
| `/budget/check` | POST | Evaluate budget level, auto-pause if exceeded. |
